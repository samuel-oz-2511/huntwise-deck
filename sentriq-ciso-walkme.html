How Sentriq works
Layer 1

Behavioral Baseline

Every application submission carries a fingerprint: how the form was filled, tools used, timing patterns, device signals. Sentriq reads these at submission, before anyone reviews the candidate.
  • Tool and automation fingerprinting
  • Behavioral anomaly detection
  • Device and environment signals
  • Timing and cadence patterns
Layer 2

Trust Layer

Sentriq does not evaluate each application in isolation. Every submission contributes to a cross-company signal graph that builds across all applications, across all companies on the network.
  • Cross-application pattern matching
  • Network-wide anomaly scoring
  • Shared fraud intelligence across companies
  • Signal sharpens with every application processed
Layer 3

Async Personalized Challenge

Flagged applications trigger a personalized async challenge sent to the applicant. It does not test skills. It helps understand whether the flag is a real gap or concern before any recruiter time is spent.
  • Tailored to the specific application and role
  • Invisible friction for legitimate candidates
  • No recruiter involvement required
  • Result surfaces in Lever alongside the candidate
What you see
Each application arrives in Lever with a trust signal. High-risk submissions trigger a CISO-level alert before any recruiter time is spent.
Normal: proceed as usual. Flagged: confidence indicator surfaces before the phone screen. High-risk: parallel alert to your security team before the pipeline advances.
The Cloudflare analogy
Like Cloudflare, Sentriq sits upstream of your entire stack.
Cloudflare does not inspect each request in isolation. It draws on network-wide traffic intelligence built across millions of endpoints. Sentriq works the same way: behavioral patterns, tool fingerprints, and anomaly clusters accumulate across every application processed, across every company on the network. The unit of analysis is the application, not the candidate. A single candidate may submit multiple applications. Each one is evaluated independently against everything Sentriq has seen. No changes to Lever, no recruiter workflow changes, no PII stored.
