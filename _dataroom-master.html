04
Product: Candidate Hiring Captcha
ALL
Critical language rules: "Trust signal" or "trust signals" : not "trust score". "Platform-agnostic": not "ATS-agnostic". The product surfaces signals. Humans decide. Zero auto-rejections. Every flag is a human decision.
1
Behavioral Auto-Flag
Passive: runs on every application
Behavioral pattern analysis. Detects Application Signal Manipulation: keyword injection, hidden text, ATS-optimized formatting. Flags claimed skills with no supporting timeline, title-ability mismatches. No anomaly: passes through with no delay. Anomaly detected: Layer 2 triggered. Candidate sees nothing.
2
Behavioral Challenge
Active: triggers on anomaly from Layer 1. ~$10/challenge average.
Short task framed as part of the application. Calibrated to the specific profile the candidate presented. Scores behavioral signals and content quality. Detects: Behavioral Spoofing (AI-assisted response generation, copy-paste patterns mimicking organic typing, response timing anomalies). Consistent with profile: cleared. Significant divergence: Layer 3 triggered. Volume: ~200 challenges/month per customer at steady state.
3
Human Review
Human-in-loop: triggers for 5-10% of challenges. $25-50/case reviewed.
Fraud analyst review of Layer 2 outputs. Signal interpretation and employer escalation support. Detects Identity Fabrication: synthetic CV generation, deepfake profile photos, stolen credential packages, VPN-masked location spoofing. EU AI Act Article 6, Annex III, point 4 mandates human oversight for AI in hiring : this transforms regulatory burden into sales trigger. Required by design, not optional.
Output
Trust signal per application delivered as an evidence card. Shows what was detected and why. No black box. Human decides. Evidence card appears in the recruiter's existing workflow, where they already work. Integration is platform-agnostic: upstream of the ATS, after application submission form, before application enters pipeline.
For HR / TA Leaders
Frame as: Candidate Hiring Captcha. Invisible to legitimate candidates. Surfaces anomalies as one clear signal. Lifecycle diagram position: between application submission and ATS intake. Cloudflare analogy: the application never reaches your ATS without passing through Sentriq first.
For CISOs
Frame as: shift-left hiring security. Hiring funnel is an attack surface. DPRK operatives, credential thieves, proxy candidates all enter through the application layer. Sentriq is the pre-ATS detection layer , same architectural position as WAF before the web server. CISO budget, security risk frame, not HR software.
05
Structural Moat
ALL
1. ATS Contractual Blockade
Greenhouse, Workday, Lever, Ashby all operate under customer agreements that prohibit using one employer's candidate data for another employer's benefit. This is a structural data privacy and commercial obligation. It cannot be waived. Any ATS attempting to build a cross-employer fraud graph would be in breach of contract with every customer simultaneously. This is not a feature gap. It is a legal wall that cannot be removed with capital.
2. Behavioral Corpus: Huntwise Flywheel
Sentriq's detection signatures are derived from 18 months of live operation inside real ATS systems via Huntwise (100 beta users, live applications, real companies). Identity vendors (Persona, Checkr) have no behavioral baseline. DPRK operatives pass identity checks using real stolen SSNs. Starting a competitor today requires building the attack tool first (which no enterprise security company can credibly do), then operating it for 18 months. Timeline cannot be compressed with capital alone.
3. Automation Defeat: The Last Layer Standing
Camoufox (7,400+ GitHub stars, ~1M PyPI downloads/month) defeats IP blocking, browser fingerprinting, and form-level detection. It is integrated into NousResearch's Hermes Agent and OpenClaw (most-starred GitHub project as of April 2026). What Camoufox cannot defeat: a behavioral challenge calibrated to a specific claimed profile (content cannot be faked without knowing the calibration target) and a cross-company graph (campaign-level patterns survive proxy rotation). The moat is the last detection layer standing after automation defeats everything else.
4. CISO Budget Capture
DPRK incidents and FBI/DOJ/Senate attention made hiring fraud a security risk budget item in 2024. The budget lives with the CISO, not HR. Same budget line as Crowdstrike, SentinelOne, Persona. This is not a positioning choice. It is a documented market shift that existing recruiting platforms cannot follow (wrong buyer, wrong budget, different procurement cycle).
When investors name Persona: It is a diligence test. Answer: Persona answers "is this person real?" Sentriq answers "is this application real?" Identity verification is orthogonal to application authenticity. DPRK operatives use real stolen SSNs. They pass Persona. They fail Sentriq's behavioral challenge. "You have Persona, you know the person is real. Sentriq tells you if the application is real." Complement, not competitor.

Classification

August 2, 2026. This is a hard external deadline that creates mandatory purchase events.
Requirements

BIPA Definition (Illinois): Reference Only
"any information, regardless of how it is captured, converted, stored, or shared, based on an individual's biometric identifier (retina or iris scan, fingerprint, voiceprint, scan of hand or face geometry) used to identify an individual"
Sentriq collects behavioral metadata, not biometric identifiers as defined above. This is the regulatory distinction documented in the legal opinion.
